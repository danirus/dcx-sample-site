django-comments-xtd is a reusable Django app that extends django-contrib-comments with thread support, following-up notifications and comments that only hit the database after users confirm them by email.

Threaded comments

Comments can be nested up to a maximum thread level. The maximum thread level is customizable either for the whole site or on a per app.model basis.

Notifications and mute-links

Comment authors may opt to receive notifications of follow-up comments, and the notifications can be cancelled at any time by clicking on the mute-link that comes with the follow-up notifications.

Comment confirmation

Comment confirmation via email when comment authors are not authenticated. Comments hit the database only after they have been confirmed.

Comment reactions

Enable comment reactions starting with the classic like and dislike buttons and extend them to any set of reactions that may suit your audience.

Template tags

Comes with template tags and filters to retrieve comments sorted and/or filtered by nested comments, reactions or publish date. And easy to extend with Django's queryset.

CSS and JavaScript

It does not depend on any heavy frontend framework. Comes with fully responsive CSS design, and a frontend written in vanilla JavaScript. Read the docs to extend it with your own frontend choices.

by Suzy Hazelwood

Two apps using django-comments-xtd

Within the same Django project, two apps use django-comments-xtd in different ways. The stories app and the quotes app.

Stories app

Comments for the stories app can be nested up to second level. Comment reactions are enabled and extended ala GitHub. Both, comments and reactions are sent using vanilla JavaScript.

Read my stories.

Quotes app

The quotes app is pure backend vintage app, it does not use JavaScript. But still comments can be nested, here up to first level, could be more. Reactions are disabled but can be enabled.
